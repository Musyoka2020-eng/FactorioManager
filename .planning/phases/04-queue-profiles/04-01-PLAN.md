<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk config and profile files → core services | Untrusted local JSON is loaded into queue/profile state |
| Core diff builder → later UI dialogs | Core-generated action lists drive destructive enable/disable/apply behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `mod-list.json` and profile JSON parsing | mitigate | Validate JSON shape, default safely on malformed input, and write via temp-file replace to avoid partial corruption |
| T-04-02 | Repudiation | Undo snapshot identity | mitigate | Persist timestamped snapshot IDs and bind them to one profile-apply operation so later undo restores the exact pre-apply state |
| T-04-03 | Denial of service | Oversized queue/profile payloads | accept | Desktop-local files are low-volume in this phase; keep JSON structures bounded and avoid recursive loading |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local mod inventory and `mod-list.json` → Checker UI | File-derived enabled state becomes user-editable table controls |
| Saved profile names and preset labels → dialogs | Persisted text crosses into modal UI surfaces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Tampering | Enabled-state writes to `mod-list.json` | mitigate | Route all toggles through `ModListStore` atomic writes and never perform direct ad hoc JSON edits in the tab |
| T-04-11 | Information disclosure | Profile names and preset labels in dialogs | mitigate | Display all user-authored or preset text through plain-text Qt labels and line edits only |
| T-04-12 | Denial of service | Repeated update enqueue actions | mitigate | Disable duplicate queue submissions per selection while an equivalent queued or running update item already exists |
</threat_model>

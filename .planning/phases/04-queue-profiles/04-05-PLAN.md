<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Profile diff payload → destructive apply path | Previewed actions become real enabled-state and queue mutations |
| Snapshot token → undo restore | One-click undo restores prior state from persisted snapshot data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Tampering | Diff preview vs executed actions | mitigate | Execute from the immutable diff payload generated before confirm rather than recomputing state during execution |
| T-04-14 | Repudiation | Undo restoring the wrong snapshot | mitigate | Bind undo to a single stored snapshot ID attached to the completed queue item and invalidate it on the next profile apply |
| T-04-15 | Information disclosure | Inspect and diff text rendering | mitigate | Render diff rows and inspect data with Qt plain-text widgets and monospace only where the UI-SPEC explicitly allows it |
</threat_model>

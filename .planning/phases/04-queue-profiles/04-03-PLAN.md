<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-entered URL/name → queued download job | Untrusted input becomes a long-running queue operation |
| Download job failure payloads → inspect drawer | Runtime exceptions are surfaced back into UI actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering | Download queue metadata | mitigate | Validate and normalize URL or mod-name input before enqueueing and store plain-text inspect payloads only |
| T-04-08 | Denial of service | Duplicate download job starts on retry/resume | mitigate | Centralize lifecycle in `DownloadQueueJob` plus controller-owned state guards so one operation cannot spawn multiple active workers |
| T-04-09 | Information disclosure | Inspect details leaking rich text or HTML | mitigate | Reuse plain-text progress/error rendering and never render inspect text as rich text |
</threat_model>

def _on_clean_backups_clicked(self):

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Mods folder input → filesystem read/delete | Checker reads .zip files; delete/backup operations write/delete local files |
| Mod names from filesystem → QTableWidget display (HTML) | Mod names from disk rendered in HTML operation log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | Delete/Clean Backups confirmation (persistent toast) | mitigate | Operations only execute after user explicitly clicks "Delete" in persistent toast; "Cancel" is always present |
| T-06-02 | Information Disclosure | HTML injection via mod name in operation log | mitigate | Apply `html.escape(message)` before embedding mod names in `<span>` HTML in operation log |
| T-06-03 | Tampering | Arbitrary path traversal via mods folder | accept | Folder path is user-selected via native OS dialog; same risk profile as Tkinter version; no path inputs accepted from untrusted sources |
| T-06-04 | DoS | Concurrent worker operations | mitigate | Store active worker in `self._active_worker`; disable action buttons during operation; re-enable on completion signal |
</threat_model>

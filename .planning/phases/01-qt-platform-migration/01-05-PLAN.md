**DownloadWorker (QThread subclass):**

Signals:
- `progress = Signal(int, int)` — (completed_count, total_count)
- `mod_status = Signal(str, str)` — (mod_name, status_text) for sidebar rows
- `log_message = Signal(str, str)` — (message, level_name) for progress console
- `finished = Signal(bool, list)` — (all_succeeded, failed_mod_names)

Constructor stores all parameters in `__init__` before `start()` is called:
- `mod_url: str`
- `mods_folder: str`
- `include_optional: bool`

`run()` method:
1. Creates `ModDownloader()` instance
2. Calls `downloader.download_mod(url, folder, progress_callback=_on_progress, per_mod_callback=_on_mod_status)`
3. `_on_progress(completed, total)` emits `self.progress.emit(completed, total)` and `self.log_message.emit(f"Downloading: {completed}/{total} mods ({int(completed/total*100)}%)", "INFO")`
4. `_on_mod_status(mod_name, status)` emits `self.mod_status.emit(mod_name, status)`
5. On `PortalAPIError` (offline/not found): emits `self.log_message.emit(str(e), "ERROR")` and `self.finished.emit(False, [])`
6. On success: emits `self.finished.emit(len(failed) == 0, failed)`

Also create ResolveWorker for the "Load Mod" button (full dependency resolution):
- Signals: `resolved = Signal(dict)` (mod info dict), `error = Signal(str)`
- Constructor: `mod_url: str`
- `run()`: calls `FactorioPortalAPI().get_mod_info()` or `get_mod_by_url()`, catches exceptions, emits resolved or error

Also create SearchWorker for URL field debounce auto-search (500ms after typing stops):
- Signals: `result = Signal(dict)`, `error = Signal(str)`
- Constructor: `query: str`
- `run()`: calls portal API, emits result or error

All three worker classes go in this file. DownloaderTab class written in Task 2.

|----------|-------------|
| URL input field → FactorioPortalAPI | User-typed URL sent to external API |
| Mods folder path → filesystem write | Downloaded .zip files written to user-selected directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Mod URL input | mitigate | `validate_mod_url()` from utils/helpers.py is already called by existing downloader logic before any network call; no additional validation needed at UI layer |
| T-05-02 | Information Disclosure | HTML injection via mod name in progress console | mitigate | Apply `html.escape(message)` before embedding mod names in `<span>` HTML in progress console |
| T-05-03 | Tampering | Arbitrary filesystem write via mods folder picker | accept | User explicitly selects folder via native OS dialog; downloads go only to user-selected path; same risk as Tkinter version |
| T-05-04 | DoS | Concurrent download worker creation | accept | Download button is disabled during active download; only one worker can run at a time |
</threat_model>

# Phase 1 — UI Design Contract: Qt Platform Migration and Behavior Parity

> Visual and interaction contract for the PySide6 Qt migration.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Desktop-adapted spec.** All fields are expressed in Qt/PySide6 terms.
> No web frameworks, no shadcn, no npm registries apply.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | QSS (Qt Style Sheets) — single global stylesheet | D-09 |
| Preset | Not applicable (desktop/Qt) | — |
| Widget library | PySide6 built-in widgets (`QMainWindow`, `QTabWidget`, `QWidget`, `QLabel`, `QPushButton`, `QLineEdit`, `QProgressBar`, `QTableWidget`, `QTextEdit`, `QSplitter`, `QFrame`, `QCheckBox`, `QRadioButton`, `QScrollArea`) | D-11 |
| Icon strategy | Unicode glyphs inline in label/button text; `QStyle.StandardPixmap` for native OS dialogs (folder picker); no external icon library | PARITY-CHECKLIST.md |
| Font family | `"Segoe UI"` on Windows; system sans-serif fallback on other platforms. Set once via `QApplication.setFont(QFont("Segoe UI", 10))` | existing `main_window.py` |
| QSS file location | `factorio_mod_manager/ui/styles/dark_theme.qss` — single extractable file, structured for Phase 2 Fluent expansion | D-09 |

**Phase 2 forward-compatibility requirement:** The QSS file must use named selectors (`QPushButton#accentButton`, `QFrame#headerSeparator`, etc.) so Phase 2 can extend or override individual rules without rewriting the base stylesheet. Never use inline `setStyleSheet()` calls — all styling goes through `dark_theme.qss`.

## Spacing Scale

Expressed as `setContentsMargins()` and `setSpacing()` integer pixel values. All values are multiples of 4.

| Token | Value | Qt API | Usage |
|-------|-------|--------|-------|
| `SPACING_XS` | 4 px | `setSpacing(4)` | Icon-to-text gap; checkbox leading margin |
| `SPACING_SM` | 8 px | `setSpacing(8)` / `setContentsMargins(8,8,8,8)` | Tab bar internal padding; compact label inner padding |
| `SPACING_MD` | 12 px | `setContentsMargins(12,10,12,10)` | Notification frame padding (matches existing 12/10 tkinter); button horizontal padding |
| `SPACING_LG` | 16 px | `setContentsMargins(16,16,16,16)` | Tab page content margin; major group padding |
| `SPACING_XL` | 24 px | `setSpacing(24)` | Separation between major layout groups within a tab |
| `SPACING_2XL` | 32 px | — | Reserved; Phase 2 shell layout only |

**Explicit exceptions (not on the scale):**
- Header separator: `setFixedHeight(1)` — 1 px rule
- Status bar: `setFixedHeight(28)` — fixed for parity
- `SPACING_NOTIF_V = 10` — parity-locked from Tkinter `pady=10` (notification frame vertical padding); used in `setContentsMargins(12, 10, 12, 10)` on the toast `QFrame`
- Downloader per-mod sidebar: `setFixedWidth(220)` — column constraint
- Checker left sidebar: `setFixedWidth(220)` — column constraint
- Checker right sidebar: `setFixedWidth(280)` — column constraint
- View Details popup: `setFixedSize(1000, 800)` — from PARITY-CHECKLIST.md
- Checker operation log: `setFixedHeight(120)` minimum (or bottom split)

## Typography

`QFont` size in **points** (Qt default). Line height is Qt-managed; do not override unless wrapping is required.

| Role | Size | Weight | `QFont.Weight` constant | Usage |
|------|------|--------|------------------------|-------|
| Body | 10 pt | Normal | `QFont.Weight.Normal` | Tab content, log lines, table rows, notification message body |
| Label / UI | 9 pt | Normal | `QFont.Weight.Normal` | Field labels, column headers, button text, secondary text |
| Section heading | 11 pt | Bold | `QFont.Weight.Bold` | Tab labels, sidebar section headings |
| Window title | 14 pt | Bold | `QFont.Weight.Bold` | Main window header label |
| Notification icon | 14 pt | Bold | `QFont.Weight.Bold` | Toast glyph characters (✓, ✗, ⚠, ℹ) |
| Log / console | 10 pt | Normal | `QFont.Weight.Normal` | Monospace `QTextEdit` in Logs tab; use `QFontDatabase.systemFont(QFontDatabase.SystemFont.FixedFont)` |

**Declared weights (2 max):** Normal (`QFont.Weight.Normal`) + Bold (`QFont.Weight.Bold`). `DemiBold`/`SemiBold` is not used in this spec.

## Tab Layout Specifications

### Tab 1 — Downloader

Vertical `QVBoxLayout` with 16 px margins:

1. **URL row** — `QHBoxLayout`: `QLineEdit` (placeholder below) + `QPushButton "Load Mod"` (default style)
2. **Folder row** — `QHBoxLayout`: `QLabel "Mods Folder:"` + `QLineEdit` (read-only) + `QPushButton "Browse"` (default style)
3. **Options row** — `QHBoxLayout`: `QCheckBox "Include optional dependencies"`
4. **Download row** — `QPushButton "Download"` (accent style, full-width or right-aligned)
5. **Progress section** — `QHBoxLayout`:
   - Left `QVBoxLayout`: `QProgressBar` (full width) stacked above `QTextEdit` progress console (read-only, color-coded log)
   - Right: Per-mod sidebar — `QScrollArea` or `QListWidget`, `setFixedWidth(220)`, rows showing mod name + status text

### Tab 2 — Checker & Updates

Outer `QVBoxLayout`:

- Top: horizontal `QSplitter(Qt.Orientation.Horizontal)`:
  - **Left panel** `setFixedWidth(220)`: mods folder display + status indicator + `QVBoxLayout` action button stack (Scan, Check Updates, Update Selected, Update All, Delete, Backup, Clean Backups, View Details)
  - **Center**: `QTableWidget` (flexible) — columns: checkbox (`QCheckBox`), Name, Status, Installed → Latest, Author, Downloads
  - **Right panel** `setFixedWidth(280)`: statistics `QGroupBox` + `QLineEdit` search + status filter `QPushButton` group (All / Outdated / Up to Date / Selected) + sort `QRadioButton` group (Name / Version / Downloads / Date)
- Bottom: operation log `QTextEdit` (read-only, color-coded, `setFixedHeight(120)` minimum)

### Tab 3 — Logs

`QVBoxLayout`:
- `QTextEdit` (fill remaining space, read-only, fixed-width font, color-coded HTML log entries)
- Bottom `QHBoxLayout` right-aligned: `QPushButton "Clear Log"` (default style)

## Notification / Toast System

Implemented as `QFrame` overlay widgets parented to the `QMainWindow` central widget, positioned absolutely in code (not via layout manager). `QFrame.raise_()` called after creation to ensure overlap.

### Positioning

| Property | Value |
|----------|-------|
| Anchor | Top-right of `QMainWindow` central widget |
| Right margin | 16 px from window right edge |
| Top margin | 16 px below top of central widget (below header region) |
| Max width | 420 px |
| Stacking order | Newer toasts below existing; 8 px vertical gap between stacked toasts |
| Reposition trigger | `QMainWindow.resizeEvent` — recalculate positions on window resize |

### Visual Appearance

| Type | `QFrame` bg | Border color | Icon glyph | Icon color | QSS selector |
|------|------------|-------------|-----------|-----------|-------------|
| success | `#2d5016` | `#4ec952` | `✓` | `#4ec952` | `QFrame[notifType="success"]` |
| error | `#3a0f0f` | `#d13438` | `✗` | `#d13438` | `QFrame[notifType="error"]` |
| warning | `#3a2f1a` | `#ffad00` | `⚠` | `#ffad00` | `QFrame[notifType="warning"]` |
| info | `#1a2a3a` | `#0078d4` | `ℹ` | `#0078d4` | `QFrame[notifType="info"]` |

- `border-radius: 4px`
- `setContentsMargins(12, 10, 12, 10)` (replicates existing tkinter padx=12, pady=10)
- Icon: `QLabel`, 14 pt Bold, right margin 10 px
- Message: `QLabel`, 10 pt Normal, `#e0e0e0`, `setWordWrap(True)`, max wrap width 350 px

### Dismiss Behavior

| Type | Duration | Dismiss mechanism |
|------|----------|-------------------|
| Auto-dismiss | 4000 ms | `QPropertyAnimation` on `windowOpacity`, 1.0 → 0.0 over 300 ms; `deleteLater()` on `finished` |
| Persistent | 0 ms (no timer) | User clicks ×, or action button ("Delete" / "Cancel") inside the frame |

### Persistent + Action Buttons

For Delete confirmation and Clean Backups confirmation: add a `QHBoxLayout` with action `QPushButton`s inside the notification `QFrame`. "Delete" uses destructive style; "Cancel" uses default style. Clicking either dismisses the toast immediately and fires the corresponding callback.

## Log Line Color Contract

All color-coded log display is implemented via `insertHtml()` in read-only `QTextEdit` widgets. Use `QTextCursor` + `QTextCharFormat` for performance, or format each line as `<span style="color:{hex};">{message}</span><br>`.

| Log Level | Color Hex | Display |
|-----------|----------|---------|
| INFO | `#0078d4` | Blue |
| DEBUG | `#b0b0b0` | Gray |
| WARNING | `#ffad00` | Yellow |
| ERROR / CRITICAL | `#d13438` | Red |
| SUCCESS (custom) | `#4ec952` | Green |

*Source: D-10 (CONTEXT.md) + existing Tkinter progress console behavior.*

Auto-scroll after each append:
```python
sb = text_edit.verticalScrollBar()
sb.setValue(sb.maximum())
```

## Debounce and Timing Contracts

| Interaction | Delay | Implementation |
|------------|-------|---------------|
| URL field auto-search | 500 ms after last keystroke | `QTimer.singleShot(500, ...)` reset (cancel + restart) on each `textChanged` signal |
| Auto-scan on first Checker tab open | ~3 s after tab activated | `QTimer.singleShot(3000, self._run_scan)` — fires once via `self._auto_scan_done` flag |
| Duplicate auto-scan guard | One-time | `self._auto_scan_done: bool = False`; set `True` after first auto-scan fires; `tabChanged` signal checks flag |

## QSS File Structure

`factorio_mod_manager/ui/styles/dark_theme.qss` — sectioned for Phase 2 override safety:

```qss
/* === 1. GLOBAL BASE === */
/* QWidget, QMainWindow, font */

/* === 2. SURFACES === */
/* QFrame, QGroupBox panel variants */

/* === 3. INPUTS === */
/* QLineEdit, QTextEdit, QComboBox */

/* === 4. BUTTONS === */
/* QPushButton (default), #accentButton, #destructiveButton */

/* === 5. TABLES === */
/* QTableWidget, QHeaderView::section */

/* === 6. TABS === */
/* QTabWidget::pane, QTabBar::tab, QTabBar::tab:selected */

/* === 7. PROGRESS === */
/* QProgressBar, QProgressBar::chunk */

/* === 8. SCROLLBARS === */
/* QScrollBar (minimal dark style) */

/* === 9. STATUS BAR === */
/* QStatusBar */

/* === 10. NOTIFICATIONS === */
/* QFrame[notifType="success|error|warning|info"] */
```

Object naming convention: `widget.setObjectName("accentButton")` — QSS selects via `QPushButton#accentButton { ... }`. This is the extension point for Phase 2.

## Component Contracts

### 1. Global Search Bar (`GlobalSearchBar`)

**Location:** Header utility zone — right side of `#headerWidget`, left of future settings entry point.

**Widget type:** `QLineEdit` with custom `#globalSearchBar` object name.

**QSS spec:**
```
QLineEdit#globalSearchBar {
    background-color: {BG_INPUT};
    border: 1px solid {BORDER_DEFAULT};
    border-radius: 4px;
    padding: 4px 12px;
    min-width: 200px;
    max-width: 300px;
    color: {FG_PRIMARY};
}
QLineEdit#globalSearchBar:focus {
    border: 1px solid {BORDER_FOCUS};
}
```

**Placeholder text:** `"Search mods…  Ctrl+K"`

**Keyboard shortcut:** `QShortcut(QKeySequence("Ctrl+K"), main_window)` focuses the bar.

**Debounce:** 350 ms timer (`QTimer.singleShot`) before firing search — matches existing `SearchWorker` pattern in `DownloaderTab`.

---

### 2. Search Results Popup (`SearchResultsPopup`)

**Widget type:** `QFrame` popup, `Qt.WindowType.Popup` flag, anchored below `GlobalSearchBar`.

**Size:** 400px wide × auto-height (max 480px with scroll), aligned to leading edge of search bar.

**Sections within popup (grouped by source):**

| Group header | Content rows |
|---|---|
| `"Installed"` | Mod name + status badge |
| `"Queued"` | Mod name + queue position |
| `"Portal"` | Mod name + download count |

Group header style: `FG_SECONDARY`, 9pt, `SPACING_SM` vertical padding, `BORDER_DEFAULT` bottom separator.

Result row style: `FG_PRIMARY`, `SPACING_SM` padding, `BTN_HOVER_BG` on hover.

**Empty state:** Single centered label `"No results for '{query}'"` in `FG_SECONDARY`.

**Selection behavior (D-05):** Clicking any result fires `result_selected(mod_name, source)` signal — caller opens a `ModDetailsDialog` (not page navigation).

**Dismiss:** Popup closes on focus-out, Escape key, or result selection.

---

### 3. Shared Filter/Sort Bar (`FilterSortBar`)

**Widget type:** `QWidget` subclass, horizontal layout.

**Composition:**
- `QLineEdit` search input (placeholder `"Filter mods…"`, 200px min-width)
- `QComboBox` status filter (options: `"All"`, `"Up to date"`, `"Outdated"`, `"Selected"`)
- `QComboBox` sort-by (options: `"Name"`, `"Version"`, `"Downloads"`, `"Date"`)
- Optional `QComboBox` priority filter (context-specific: injected per page — see below)

**Spacing:** `SPACING_SM` between controls.

**Context-specific extras (injected by host page, not part of base bar):**

| Page | Extra control |
|---|---|
| CheckerTab | Priority combo: `"Any priority"`, `"Outdated"`, `"Selected"`, `"Errors"` |
| DownloaderTab | Category chips row (see §4) — placed above the bar |

**Signal:** `filter_changed(query: str, status: str, sort: str, priority: str)` emitted on any change with 200 ms debounce.

**Extraction requirement:** `CheckerPresenter.filter_mods(...)` existing signature must remain compatible. `FilterSortBar` calls it via `checker_presenter.filter_mods(mods, query, filter_mode, selected_mods, sort_by)`.

---

### 4. Category Chips (`CategoryChipsBar`)

**Widget type:** `QWidget` with `QHBoxLayout` + `QScrollArea` (horizontal scroll, no vertical scroll).

**Location:** Above `FilterSortBar` inside `DownloaderTab`, below the page header zone.

**Chip widget:** `QPushButton` with `#categoryChip` object name.

**QSS spec:**
```
QPushButton#categoryChip {
    background-color: {BG_PANEL};
    color: {FG_SECONDARY};
    border: 1px solid {BORDER_DEFAULT};
    border-radius: 12px;
    padding: 3px 10px;
    font-size: 9pt;
}
QPushButton#categoryChip:hover {
    background-color: {BTN_HOVER_BG};
    color: {FG_PRIMARY};
}
QPushButton#categoryChip[selected="true"] {
    background-color: {ACCENT};
    color: #ffffff;
    border: 1px solid {ACCENT};
}
```

**Initial chips:** `["All"]` + top categories from portal API. Populated by `SearchWorker`-equivalent thread on tab load.

**Behavior:** Single-select toggle. Selecting a chip calls `portal.search_mods(category=chip_name)`. `"All"` clears category filter.

---

### 5. Settings Page (`SettingsPage`)

**Location:** New page in `QStackedWidget` (`page_host`). Nav rail entry added at bottom of nav rail with icon `"⚙"` and label `"Settings"`.

**Also accessible from:** Header utility zone — `QPushButton#settingsButton` (gear icon `"⚙"`, `24×24px`) placed right of `GlobalSearchBar`.

**Page structure:** `QScrollArea` containing a single `QWidget` with `QVBoxLayout`. Groups as `QGroupBox`:

| Section | Controls |
|---|---|
| **Paths** | `mods_folder`: `QLineEdit` (read-only display) + `QPushButton "Browse…"` |
| **Behavior** | `max_workers`: `QSpinBox` (range 1–16, step 1) · `auto_backup`: `QCheckBox "Auto-backup before updates"` · `auto_refresh`: `QCheckBox "Refresh on launch"` |
| **Appearance** | `theme`: `QComboBox` with options `["Dark", "Light", "System"]` |
| **Advanced** | (reserved — empty `QGroupBox` with `"No advanced settings in this version"` label) |

**Form layout:** `QFormLayout` inside each `QGroupBox` — label column 140px fixed, field column stretches.

**Footer buttons (outside scroll area, pinned to page bottom):**
- `QPushButton#accentButton "Save"` — applies and persists all changes
- `QPushButton "Reset to Defaults"` — restores `Config.DEFAULTS` values (non-destructive for paths)
- `QPushButton "Cancel"` — reverts unsaved edits, navigates back to previous page

**Save behavior (D-15):** No per-field auto-commit. All fields write to `Config` only when "Save" is clicked.

**Theme apply (D-17):** On "Save", if theme changed: call `load_and_apply_theme(new_theme)` immediately — user sees change before dialog closes.

---

### 6. Theme Toggle Logic

**Theme options:** `"dark"` → loads `dark_theme.qss`, `"light"` → loads `light_theme.qss` (new; Phase 3 creates minimal light variant), `"system"` → detects OS preference and loads matching file.

**System mode (D-18):** Connect `QApplication.instance().paletteChanged` signal → re-evaluate OS preference → hot-swap QSS if changed.

**Token file:** `tokens.py` extended with `LIGHT_*` token variants. `load_and_apply_theme()` utility in `factorio_mod_manager/ui/styles/` selects the correct token set and formats the QSS template.

---

### 7. Mod Details Popup (`ModDetailsDialog`)

**Widget type:** `QDialog`, modal, min-size `520×400`.

**Triggered by:** Global search result selection (D-05). Also reusable from checkerTab row double-click (future-safe).

**Content:** Mod name (`FG_PRIMARY`, 12pt bold), author (`FG_SECONDARY`, 9pt), version, status badge, description text area (`QTextEdit` read-only), portal link button.

**Footer:** `QPushButton "Close"` (standard) and context-sensitive action button (e.g., `"Add to Queue"` or `"Update"` based on source).

## Interaction Patterns

### Unsaved Settings Guard
If user navigates away from Settings page with unsaved changes: show `QMessageBox.question` with "Save", "Discard", "Cancel". Only destructive-confirmation pattern allowed per Phase 2 D-12.

### Theme Change Preview
Theme change is previewed **immediately** on `QComboBox.currentIndexChanged` without saving — user sees the change live. Pressing Cancel reverts to prior theme.

### Search Keyboard Flow
- `Ctrl+K` → focus `GlobalSearchBar`
- Typing → debounce 350 ms → populate `SearchResultsPopup`
- `↑`/`↓` → navigate results
- `Enter` → select focused result → open `ModDetailsDialog`
- `Escape` → close popup, return focus to previous widget

### Category Chip Selection
- Single chip active at a time
- Selecting a chip sets `[selected="true"]` Qt property and fires portal query via `SearchWorker`
- Portal query results replace the existing downloader result list (not appended)

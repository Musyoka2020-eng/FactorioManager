<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
All Phase 3 components integrated:
- Style foundation: dark_theme.qss extended, load_and_apply_theme(), light_theme.qss (light + system switching)
- Credential hardening: username/token removed from config DEFAULTS and portal API
- FilterSortBar: in CheckerTab right sidebar replacing inline filter controls (SRCH-02)
- CategoryChipsBar: in DownloaderTab above stage-1 results (SRCH-03)
- SettingsPage: accessible via nav rail "⚙ Settings" and header settings button (SETT-01, SETT-02, SETT-03)
- GlobalSearchBar: in header with Ctrl+K focus, debounced popup (SRCH-01)
- ModDetailsDialog: opens on popup result click
  </what-built>
  <how-to-verify>
Start the application: `python -m factorio_mod_manager` (or launch the built exe).

**SRCH-01 — Global Search**
1. Press Ctrl+K — cursor should jump to the search bar in the header
2. Type a partial mod name (e.g. "bob") — popup should appear below the search bar within ~350ms
3. Portal results section should appear after ~1s (network call)
4. Click a portal result — ModDetailsDialog should open with mod name, author, description
5. Close the dialog; press Escape in the search popup — popup should dismiss
6. Verify search input clears after result selected

**SRCH-02 — Filter / Sort Bar (Checker tab)**
1. Click the "🔍 Checker" nav item
2. Confirm the right sidebar now shows FilterSortBar (not the old inline filter radio buttons)
3. Scan mods (if any installed) — confirm filter buttons (All / Outdated / Up to date) work
4. Confirm sort dropdown (Name / Status / Downloads) sorts the table correctly
5. Confirm the search field in FilterSortBar filters the list

**SRCH-03 — Category Chips (Downloader tab)**
1. Click the "📥 Download" nav item
2. Confirm CategoryChipsBar appears above the search/results area
3. Click a category chip (e.g. "Logistics") — confirm the list updates with portal results for that category
4. Click the chip again to deselect — confirm "All" state restored

**SETT-01 — Settings Page Reachability**
1. Click the header "⚙" button — settings page should open (nav rail should highlight Settings)
2. Click "⚙  Settings" in the nav rail — same result
3. Verify all settings fields visible: Mods Folder (path picker), Theme dropdown, Max Workers spinner, Auto Backup toggle, Auto Check on Startup toggle

**SETT-02 — Theme Selector**
1. Open Settings
2. Change theme to "Light" → click Save → UI should re-render with light theme immediately
3. Change theme to "Dark" → Save → reverts to dark
4. Change theme to "System" → Save → app should match your OS's light/dark preference
5. Change OS light/dark mode (if possible) → app should auto-update without restart

**SETT-03 — Additional Settings**
1. In Settings, change Max Workers to 8 → Save → reopen Settings → value persists as 8
2. Toggle "Auto Backup" → Save → reopen Settings → toggle state persists
3. Click "Cancel" without saving — changes discarded, return to previous page
4. Make a change and click the "🔍 Checker" nav item — QMessageBox should appear asking Save / Discard / Cancel
5. Click Cancel in the dialog — stay on Settings page with changes intact

**Regression checks**
1. Download tab search still works (type a query, results appear)
2. Checker scan still works (if mods present): scan completes, table populates
3. Log tab shows output
4. App exits cleanly (no uncaught exceptions in terminal)
  </how-to-verify>
